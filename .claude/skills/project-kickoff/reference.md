# Project Kickoff Reference

## Agent Assignment Matrix

| Task Type | Primary Agent | Support Agent |
|-----------|--------------|---------------|
| Build software / write code | software-engineer | qa-evaluator |
| Research / analysis / data gathering | researcher | brand-strategist |
| Social media content / strategy | social-media-manager | content-creator |
| Automation / workflows / integrations | automation-architect | software-engineer |
| Business strategy / pricing / growth | brand-strategist | researcher |
| Project coordination / multi-agent work | project-orchestrator | all relevant |
| Code review / QA / testing | qa-evaluator | software-engineer |
| Blog / email / marketing copy | content-creator | social-media-manager |

## Size Estimation Guide

| Size | Duration | Example |
|------|----------|---------|
| S | < 30 min | Fix a bug, add a config, write a test |
| M | 30 min - 2 hr | Build a feature, create a component, write a blog post |
| L | 2+ hr | Design a system, build an integration, full content campaign |

## Phase Template

### Phase 1: Foundation
- Project structure and scaffolding
- Environment setup
- Core dependencies
- Database schema (if applicable)

### Phase 2: Core Build
- Primary features
- Business logic
- Integrations
- Run in parallel where no dependencies exist

### Phase 3: Quality
- Testing (unit, integration, e2e)
- Code review
- Security audit
- Performance check

### Phase 4: Ship
- Documentation
- Deployment configuration
- Final QA pass
- Monitoring setup

## Risk Categories
1. **Technical risk** — Unfamiliar tech, complex integration, performance uncertainty
2. **Scope risk** — Requirements unclear, scope creep potential
3. **Dependency risk** — External APIs, third-party services, team availability
4. **Timeline risk** — Ambitious deadlines, sequential bottlenecks
